SupplyGuard JP オンボーディングガイド

サプライチェーンセキュリティ評価CLI — 5分でセットアップ完了

STEP 1

動作環境の確認

SupplyGuard JP は Python 3.9 以上が必要です。

python --version
# Python 3.9.x 以上であれば OK
STEP 2

リポジトリのクローンとインストール

git clone https://github.com/yorosiku008/supplyguard-cli.git
cd supplyguard-cli
pip install -r requirements.txt
依存パッケージは rich（ターミナル描画）のみです。AWS不要で動作します。
STEP 3

デモモードで動作確認

3社のサンプルベンダーデータで評価フローを確認できます。

python main.py --demo
ベンダー別スコア表とグレードが表示されれば成功です。
STEP 4

3軸スコアリング — 評価内容

EU CRA・経産省ガイドラインに基づく3軸でベンダーリスクを評価します。

SBOM / CVE
45%
SBOMの有無 / CRITICAL・HIGH CVE件数
アンケート
35%
セキュリティアンケートの回答率・適合率
インシデント履歴
20%
直近12ヶ月のインシデント件数・侵害履歴

グレード基準

グレードスコア評価・推奨アクション
S90〜100優秀 — 優先取引先として継続
A80〜89良好 — 軽微な改善要求
B70〜79普通 — 改善計画の提出を要求
C60〜69要注意 — 期限付き改善を要求
D50〜59不良 — 取引条件の見直しを検討
E0〜49危険 — 取引停止・代替先検討
STEP 5

EU CRA対応チェックポイント

2027年施行予定のEU Cyber Resilience Act（CRA）に向けた確認項目：

要件チェック内容SupplyGuardでの対応
SBOM提出義務CycloneDX / SPDX形式のSBOMが存在するかSBOM軸で評価（未提出でスコア大幅減）
脆弱性開示CRITICAL CVEの修正期限遵守CVE件数と重要度を定量評価
インシデント報告重大インシデントの24時間以内報告インシデント履歴軸で評価
STEP 6

Markdownレポートの出力

python main.py --demo --output-md
# SupplyGuard_report_YYYYMMDD.md が保存されます
レポートには全ベンダーのスコア表と改善項目一覧が含まれます。監査対応資料として活用できます。
